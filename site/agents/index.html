Agent orchestration

Shared truth for fleets that never sit still.

Agents can run anywhere and still agree on who owns a task, who is leader, and when abandoned work is safe to reclaim. OctoStore provides the coordination layer without taking over execution.

Choose the smallest primitive that settles the argument.

Most agent collisions are one of two problems. A task needs one temporary owner, or a fleet needs one current leader. OctoStore implements both as expiring leases with monotonic fencing terms.

Remote leader election, zero signup.

Open a random room and let every candidate campaign. No user record is created. No API key is sent. The winner receives an opaque leader capability; followers receive the same current leader, fencing term, and retry delay.

shell · elect a coordinator
ROOM=$(curl -s -X POST https://api.octostore.io/elections \
  | jq -r .election_id)

result=$(curl -s -X POST \
  "https://api.octostore.io/elections/$ROOM/campaign" \
  -H "Content-Type: application/json" \
  -d '{
    "candidate_id":"dispatcher-west",
    "ttl_seconds":30,
    "metadata":"fleet=support-agents"
  }')

# status=leader: save leader_token and renew at renew_after_ms
# status=follower: wait retry_after_ms, then campaign again

Capability, not identity

The election room is intentionally public to anyone who knows its ID. Only the unguessable leader token can renew or resign the current term. Use generated room IDs, treat leader tokens as secrets, and self-host when the coordination namespace itself must be private.

Claim work before tools touch it.

For durable task ownership, give each task a deterministic lock name. Authenticated agents race to acquire it. One gets a lease ID; the rest observe that it is held and move on.

shell · claim task 1842
curl -s -X POST \
  https://your-octostore.internal/locks/github-issue-1842/acquire \
  -H "Authorization: Bearer $OCTOSTORE_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "ttl_seconds":120,
    "metadata":"agent=atlas run=https://trace.example/7f2a"
  }'

# acquired → work, renew, release
# held     → skip or retry

The operating loop.

Acquire

Campaign for fleet leadership or acquire a task-specific lock before starting side effects.

Fence

Carry the returned term into downstream writes. A resource that remembers the newest term can reject stale agents after partitions.

Renew

Heartbeat around half the TTL. The lease proves liveness without making a process permanent.

Release

Resign or release on clean completion. If the process vanishes, expiry is the recovery path.

What stays in your system.

OctoStore coordinates ownership. Your application still decides what work means, how tools execute, and how side effects become idempotent.

  • Keep irreversible actions behind human or policy gates.
  • Write downstream operations so a retry is safe.
  • Pass fencing terms to resources that can reject stale writers.
  • Use sessions when many ephemeral locks should disappear with one agent process.
  • Use webhooks or lock watches when operators and reconcilers need change events.

Run it where the agents run.

OctoStore is a Rust binary backed by one SQLite database in WAL mode. Start with static tokens. Add GitHub OAuth only if humans need it. Disable public elections with PUBLIC_ELECTIONS=false on private installations.
