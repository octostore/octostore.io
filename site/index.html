Leader election as a service. That's it.

This is a free thing. There's no business model. No VC funding. No enterprise tier. Just a lock service.

What it does

OctoStore provides distributed locking and leader election via a simple HTTP API. Sign up with GitHub, get a bearer token, start locking.

Get Started

  1. Authenticate with GitHub OAuth
  2. Get your bearer token
  3. Start acquiring locks

Architecture

Under the hood: a single Rust binary, Axum web framework, DashMap for concurrent in-memory storage, SQLite for user accounts. Runs on a single VM. No Kubernetes. No microservices. No blockchain.

API Overview

Auth Endpoints
# Initiate GitHub OAuth
GET /auth/github

# Rotate your bearer token
curl -X POST https://api.octostore.io/auth/token/rotate \
  -H "Authorization: Bearer YOUR_TOKEN"
Lock Operations
# Acquire a lock
curl -X POST https://api.octostore.io/locks/my-resource/acquire \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"ttl_seconds": 300}'

# Release a lock
curl -X POST https://api.octostore.io/locks/my-resource/release \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"lease_id": "uuid-here"}'

# Renew a lock
curl -X POST https://api.octostore.io/locks/my-resource/renew \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"lease_id": "uuid-here", "ttl_seconds": 300}'

# Check lock status
curl https://api.octostore.io/locks/my-resource \
  -H "Authorization: Bearer YOUR_TOKEN"

# List your active locks
curl https://api.octostore.io/locks \
  -H "Authorization: Bearer YOUR_TOKEN"

Constraints

  • 100 locks maximum per user
  • 1 hour maximum TTL
  • Ephemeral storage (we store nothing permanently)
  • Fencing tokens included for safety
