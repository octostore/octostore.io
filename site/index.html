Core stack: Skills provide capabilities. The marketplace distributes them. OctoStore coordinates execution.

The problem

Reusable skills make agents more capable. They also make collisions inevitable unless something coordinates execution ownership.

⚠️

Duplicate work

Two or more agents pick up the same task because they all have the same capability and can all see the same queue.

👀

Invisible ownership

Nobody can tell who owns a job, what they are doing, or where the current run lives.

💥

Stuck work after crashes

A worker dies and leaves the system looking busy forever unless there is lease-based recovery.

The stack

The cleanest way to describe the platform story is as three layers.

🧰

Skills

Reusable capabilities: triage issues, review PRs, generate docs, run analytics, deploy systems.

🛍️

Marketplace

Discovery and distribution for those capabilities: install, share, version, and reuse skills across teams.

🔐

OctoStore

Runtime coordination for who owns the work right now: claim, heartbeat, inspect ownership, release, recover.

What OctoStore gives you

Focused coordination instead of a heavyweight control plane.

🏆

One owner per job

Only one worker successfully claims the task.

📋

Visible ownership metadata

See who owns a job, why they claimed it, and where the run is happening.

❤️

Heartbeat while running

Keep claims alive while a worker is healthy and actively processing.

♻️

Automatic recovery

If the worker disappears, the claim expires and another worker can take over.

🔢

Fencing tokens

Use monotonic tokens to protect downstream writes from stale lock holders.

🌐

Simple HTTP API

Pure REST and JSON. Works from any language without heavy client dependencies.

The example to use everywhere

Five agents can all run the same skill on the same GitHub issue. Only one should own the job.
  1. Each agent tries to claim the same stable job name.
  2. One claim succeeds.
  3. The others can inspect the current owner.
  4. The owner heartbeats while it works.
  5. If the owner crashes, the claim expires.
  6. Another worker can take over.
