The whole API

No auth ceremony. Just campaign.

Create an opaque room, share its ID with your agents, and let each candidate make the same request. The winner receives a secret leader capability. Everyone else sees the winner and exactly when to retry.

shell · account-free election 
# 1. Open a room. No login. No token.
ROOM=$(curl -s -X POST \
  https://api.octostore.io/elections \
  | jq -r .election_id)

# 2. Every agent campaigns in the same room.
curl -s -X POST \
  "https://api.octostore.io/elections/$ROOM/campaign" \
  -H "Content-Type: application/json" \
  -d '{"candidate_id":"agent-atlas","ttl_seconds":30}'

# winner → status: leader + leader_token
# others → status: follower + retry_after_ms
Orchestration without the empire

Your agents already know how to work. Give them shared truth.

OctoStore does not execute prompts or own your workflow. It settles the hard coordination questions: who owns this task, who is leader now, whether a lease is still alive, and which fencing term is newest.

Claim

Give every issue, ticket, queue item, migration, or deployment one current owner. Leases expire when an agent disappears, so abandoned work becomes available again.

Elect

Choose one active coordinator across machines and clouds. Every term carries a monotonic fencing number so downstream systems can reject stale leaders.

Observe

Attach small operator-readable metadata, inspect current ownership over HTTP, and react to changes without coupling your fleet to a language-specific SDK.
