OctoStore examples

Simple examples that show how a distributed lock and lease service becomes a practical coordination layer.

Two local workers, one task

Two workers on one machine can both see images/thumbnail/img-123. Both try to claim it. One succeeds. The other reads the owner metadata and skips the task.

Five workers, one winner

Five workers all try to claim the same job at once. One wins. Four lose immediately. If the winner stops renewing, the claim expires and another worker can take over.

Deploy lock across CI runners

Several CI runners can all deploy production. Use deploy/prod as the stable job name so only one release runs at once.

Cross-machine cron dedup

Three servers all run the same hourly job for redundancy. Claim cron/hourly-rollup/2026-04-01T04. One wins. The others exit.

Global visibility into current work

Attach host, service, task id, claimed time, and run URL to claims. Now a status view can answer what is currently active, who owns it, and where it is running.
