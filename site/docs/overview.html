What is OctoStore?

OctoStore is a self-hostable distributed lock and lease service over HTTP. Use it locally or across many machines to coordinate work with visible ownership and automatic recovery.

Core idea

When several workers can all see the same task, one of them should claim it and the others should back off.

OctoStore provides the primitive for that:

  • create a lock or lease for a stable job name
  • renew it while the owner is alive
  • release it when done
  • let it expire automatically if the owner dies

Why this matters

Without a shared lock or lease layer, distributed systems drift into the same failures:

  • duplicate work
  • invisible ownership
  • blind retries
  • stuck work after crashes
  • ad hoc coordination logic scattered across services

OctoStore gives you one clean coordination primitive instead.

Local or global

You can use OctoStore in two natural modes.

Local coordination

Run it on one machine and let local scripts, cron jobs, workers, services, or agents coordinate through it.

Multi-machine coordination

Run one shared OctoStore deployment and let workers across many machines coordinate through the same API.

What it is not

OctoStore is not trying to be:

  • a workflow engine
  • a queue
  • a scheduler
  • a giant orchestration control plane

It is the coordination primitive underneath those higher-level patterns.
