Acquire, renew, inspect, release.

OctoStore is an HTTP API for lease-based distributed locks. The core flow is small enough to use from shell scripts and explicit enough to operate in production.

Core facts

  • Lock names are single path segments in examples: deploy-main, issue-1842.
  • Acquire with POST /locks/:name/acquire and a JSON body containing ttl_seconds plus optional metadata.
  • Acquire returns status: "acquired" when you own the lease and status: "held" when someone else does.
  • Renew returns lease_id and expires_at.
  • Release returns null on success.
export STATIC_TOKENS='alice:devtoken'
export OCTOSTORE_TOKEN='devtoken'

curl -X POST http://localhost:3000/locks/deploy-main/acquire \
  -H "Authorization: Bearer ***" \
  -H "Content-Type: application/json" \
  -d '{"ttl_seconds":60,"metadata":"local smoke test"}'
