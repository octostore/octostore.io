Local coordination with OctoStore

One of the simplest ways to use OctoStore is on a single machine. If multiple local processes can pick up the same task, run a local OctoStore server and have each process claim work through it.

When this helps

  • several agents on one box
  • multiple cron jobs
  • worker pools
  • local daemons
  • dev and demo environments

Basic pattern

  1. start a local OctoStore server
  2. choose one stable lock name per task
  3. have each worker try to claim that task
  4. renew while working
  5. release when finished
  6. let TTL expiry recover from crashes

Example metadata

{
  "owner": "thumb-worker-2",
  "host": "media-box-01",
  "service": "thumbnailer",
  "task_id": "img-123",
  "job": "generate thumbnail",
  "claimed_at": "2026-04-01T04:02:00Z"
}

Why local mode is attractive

  • dead simple
  • self-hosted
  • private
  • low latency
  • easy to debug
  • no external dependency
